# amazon-advertising-strategy

一个面向 Amazon 卖家的广告策略 skill，用来快速生成可执行的投放方案。

支持输出内容包括：

- Sponsored Products（SP）
- Sponsored Brands（SB）
- Sponsored Display（SD）
- Campaign 结构拆分
- 预算分配
- 关键词与否词策略
- ACoS / TACoS 优化节奏
- 新品广告启动方案

## 适用场景

适合以下需求：

- 不知道 Amazon 广告该怎么开始
- 广告已经在跑，但 ACoS 偏高
- 新品刚上线，需要前 30 天启动方案
- SKU 较多，需要重新拆 Campaign 和预算
- 团队需要一套统一的广告策略输出模板

## 安装

先确认本机已安装 Claude Code，并支持 skills。

```bash
npx skills add nexscope-ai/eCommerce-Skills --skill amazon-advertising-strategy -g
```

安装完成后，如果你已经能在 Claude Code 中调用 `amazon-advertising-strategy`，就说明安装成功。

## 快速开始

你只需要告诉 skill：

- 产品是什么
- 卖哪个站点
- 当前预算
- 当前广告问题
- 目标是什么

最短可用示例：

```text
请给我一套 Amazon 新品广告启动方案。产品是对讲机 PoC，美国站，每天预算 50 美元，目标是前 30 天稳定出单。
```

如果你知道 ASIN，建议一起给出：

```text
B0FHFTJPQY
请给我一套 Amazon 新品广告启动方案。产品是对讲机 PoC，美国站，每天预算 50 美元，目标是前 30 天稳定出单。
```

## 输出内容

正常情况下，skill 会输出固定 8 个部分：

1. 一、结论摘要
2. 二、现状判断
3. 三、广告结构建议
4. 四、预算分配建议
5. 五、关键词与否词策略
6. 六、ACoS / TACoS 与优化节奏
7. 七、优先级行动清单
8. 八、下一步

如果你是第一次用，建议优先看这 3 部分：

- 广告结构建议
- 预算分配建议
- 优先级行动清单

### 输出示例片段

下面是一段简化后的输出示例，方便你快速理解这个 skill 会返回什么：

```text
一、结论摘要
这款产品前 30 天应以 Sponsored Products 为主，先围绕高意图关键词完成测词和首批转化，不建议一开始就把预算铺到宽泛大词。

二、现状判断
当前链接处于新品阶段，评论基础弱，前期广告目标应优先放在拿点击、测词和筛选有效流量。

三、广告结构建议
- SP Auto：用于找词
- SP Manual Broad：用于扩词测试
- SP Manual Phrase：用于承接高相关流量
- SP Manual Exact：用于承接已验证词

四、预算分配建议
- Auto：15 美元/天
- Broad：10 美元/天
- Phrase：10 美元/天
- Exact：10 美元/天
- Product Targeting：5 美元/天
```

成熟款重启版示例片段：

```text
一、结论摘要
B07D34L57F 更适合按无线办公全尺寸键盘去打，而不是把预算主压在 bluetooth、gaming、mini 等错配词上。

二、现状判断
这条链接不是新品弱链接，而是成熟款，广告目标应放在重组预算、放大高转化词、控制泛词浪费。

三、广告结构建议
- SP-Exact-Core：承接 wireless keyboard 主流量
- SP-Exact-Feature：承接 full size / number pad / office / low profile
- SP-Product-Targeting：拦截相似办公无线键盘详情页

四、优化重点
- 把 Auto / Broad 中有效词转入 Exact
- 尽早否掉 gaming / mechanical / ipad case / mini 流量
- 30 天内沉淀核心词包、否词包和详情页清单
```

## 常见用法

新品冷启动：

```text
请给我一套 Amazon 新品广告启动方案。产品是宠物除毛刷，美国站，新品刚上架，预算每天 50 美元，目标是前 30 天完成测词和拿到第一批稳定订单。
```

ACoS 偏高：

```text
我现在 Amazon 广告 ACoS 35%，目标降到 25%。产品是厨房收纳类，共 5 个 SKU，月预算 3000 美元。请帮我重做广告结构和优化节奏。
```

从 0 到 1 搭广告结构：

```text
我刚开始做 Amazon 广告，请帮我给这款产品设计一套从 0 到 1 的广告方案。产品是便携榨汁杯，美国站，月预算 1500 美元，目标是先稳定出单。
```

成熟款重启广告结构：

```text
B07D34L57F
请帮我重做这条 Amazon 链接的广告结构。美国站，每天预算 50 美元，目标是未来 30 天找到更稳定的出单词包，并控制大词浪费。
```

## 使用建议

为了让结果更准，建议尽量提供：

- 真实预算
- 当前 ACoS / TACoS / CPC / CTR / CVR 中的任意部分
- 产品阶段
- 当前最核心的问题
- 目标

如果没有完整数据，也可以先用。至少给出产品、站点、预算和目标，就能得到第一版方案。

## 注意事项

- 这个 skill 输出的是策略和执行建议，不会自动修改 Amazon 广告后台
- 如果信息太少，结果会偏泛
- 如果能补充真实产品、竞品、关键词数据，输出会更细
- 如果用户描述的产品与真实 ASIN 数据不一致，skill 会优先按真实 ASIN 数据重写方案
- 新品前期重点通常是找词、测词、筛词，而不是过早压低 ACoS

## 验证情况

### 已验证内容

- 本地 skill 输出可稳定运行
- 可生成飞书说明文档
- 可生成基于真实产品数据的飞书实战报告文档
- 可基于真实 ASIN 数据输出定制化广告方案
- 可结合 `sorftime` 的产品、ASIN 流量词、相关关键词数据升级为更细的实战方案
- 已验证新品广告启动版方案输出
- 已验证成熟款广告重启 / 结构重做版方案输出
- 已验证当用户描述与真实 ASIN 不一致时，自动按真实数据重写方案
- 已验证实战版可进一步细化为词路优先级、否词方向、出价梯度和 30 天执行节奏

### 推荐理解方式

- 第一次使用时，可以把它理解成“Amazon 广告策略输出器”
- 如果能提供真实 ASIN、关键词或竞品数据，结果会比通用模板更细
- 如果产品其实不是新品，它也会自动切换到成熟款重启打法

### 快速判断

- 用户口述不一定等于真实产品，优先看真实 ASIN 数据
- 新品和成熟款的广告逻辑不同，输出会按真实阶段切换
- 数据越真实，结果越接近可直接执行的投放方案
